If you copy over the contents of the entire CacheIt package to your webserver, and your webserver accepts files of type .php and .phps you will be able to run the examples. If you have problems running these examples, don't worry, they're still very helpful and with the documentation, should be all you need to get up and running with CacheIt.

Please be sure to read the documentation (it really helps).

Example 1 Your basic do nothing example. In this example, the module merely returns the line of text "This is a test CacheIt." Uses module1.php and index.php
Example 2 This example was created to show you how to deal with variable input both from the user and for your module. With this example, you will be presented with a form to input your name. Once you submit the form, the following page will display "Your name is {name}". Uses module2.php and index2.php
Example 3 This example demonstrates how to use the runtime memory for displaying the same CacheIt data multiple times on the same page. Notice the 1 after the -1 in the disp_file. The 1 signifies that CacheIt should keep that data in memory until the script is completed. Uses module2.php and index2.php
Example 4 This example demonstrates the exact same concept as in example 3, BUT... it uses module2 a third time to produce different output data (and a different cache file). Uses module2.php and index2.php

 

Please note that all of these examples use a time_out of -1 to stop the module from writing out a cache file. In a production environment, you probably don't want to do this since it defeats the purpose of CacheIt. Please refer to the manual for further information.
